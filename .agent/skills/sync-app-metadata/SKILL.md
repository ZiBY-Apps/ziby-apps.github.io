# Синхронізація метаданих додатків

## When to use this skill

Цей скіл слід використовувати, коли необхідно оновити назви, підзаголовки або описи додатків на сайті, базуючись на даних з відповідних Flutter-проєктів. Також він корисний для автоматичного перекладу описів на мови, що підтримуються сайтом, та для SEO-оптимізації контенту.

## How to use it

1. **Пошук джерел**: Агент повинен перевірити кожен шлях Flutter-проєкту в воркспейсі на наявність папки `store_descriptions`.
2. **Читання даних**:
   - У папці `store_descriptions` очікуються файли формату `{lang}.json` або окремі текстові файли (наприклад, `en.txt`, `uk.txt`).
   - Якщо даних немає в цій папці, Агент може звернутися до `assets/translations/` або `pubspec.yaml` відповідного проєкту.
3. **Обробка та SEO**:
   - Для версії на сайті Агент повинен адаптувати текст для SEO (використовувати ключові слова, чіткі заклики до дії).
   - Основні тексти в самих проєктах залишаються незмінними.
4. **Локалізація**:
   - Перевірити, чи є переклади для всіх мов, які підтримує сайт (`en`, `uk`, `de`, `ru`, `es`, `fr`, `it`, `pl`).
   - Якщо переклад відсутній, Агент повинен згенерувати його на основі англійської або української версії.
5. **Оновлення сайту**:
   - Внести зміни у файл `/src/data/apps.ts` сайту.
   - Оновити динамічні SEO-метатеги, якщо це впливає на `Helmet` у компонентах.

## Validation

- Файл `src/data/apps.ts` містить актуальні дані для всіх додатків.
- Всі підтримувані сайтом мови мають відповідні переклади в об'єкті `translations`.
- Тексти описів виглядають привабливо для користувачів та оптимізовані для пошукових систем.
- Логотипи та ідентифікатори (Apple ID, Google ID) залишаються коректними.

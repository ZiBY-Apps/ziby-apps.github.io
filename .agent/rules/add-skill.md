# Правила додавання нових скілів

Ці правила є обов'язковими для дотримання при створенні будь-якого нового скіла в директорії `.agent/skills/`.

## 1. Назви та структура

- **Назва папки**: Має бути в нижньому регістрі з використанням дефісів (`-`) замість пробілів чи нижніх підкреслювань. Приклад: `my-new-skill`.
- **Обов'язковий файл**: Кожен скіл ПОВИНЕН містити файл `SKILL.md` у корені своєї папки.
- **Додаткові папки**: За потреби використовуйте `scripts/` для скриптів, `examples/` для прикладів використання та `resources/` для статичних файлів.

## 2. Формат SKILL.md

Файл `SKILL.md` повинен мати чітко визначену структуру для коректного розуміння Агентом:

### YAML Frontmatter

```yaml
---
name: my-new-skill # Унікальний ID, малі літери та дефіси (збігається з назвою папки)
description: English description of the skill. # Опис ПОВИНЕН бути англійською мовою (від 3-ї особи)
---
```

### Обов'язкові заголовки розділів

Заголовки мають бути **англійською мовою** (для системного парсингу), але контент під ними — **українською**.

1. `# Title` — Назва скіла.
2. `## When to use this skill` — Опис ситуацій, коли Агент має активувати цей скіл.
3. `## How to use it` — Покроковий алгоритм дій, використання скриптів, команд тощо.
4. `## Validation` — Критерії успішного виконання завдання за допомогою цього скіла.
5. `## Decision trees` (опціонально) — Логічні розгалуження в залежності від стану проекту.

## 3. Мовні вимоги

- Описи, коментарі та інструкції всередині розділів пишуться **українською мовою**.
- Системні ключі в YAML (**включаючи значення поля `description`**) та заголовки розділів зберігаються в англійському оригіналі.

## 4. Шляхи в скриптах

Якщо скіл використовує скрипти, шлях до них у файлі `SKILL.md` має бути повним відносно кореня проекту:
`.agent/skills/my-new-skill/scripts/my_script.sh`
